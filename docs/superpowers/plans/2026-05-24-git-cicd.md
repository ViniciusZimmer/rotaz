# Git Workflow + CI/CD Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Criar repositório GitHub privado, pipeline CI com GitHub Actions, branch protection em `main`/`dev`, e conectar Vercel com deploy automático por branch.

**Architecture:** `gh` CLI para repo e secrets; `.github/workflows/ci.yml` para lint+build em PRs; `vercel` CLI para link e env vars; branch protection via `gh api`. Dois ambientes Vercel: production (`main`) e preview estável (`dev`).

**Tech Stack:** GitHub Actions, Vercel CLI, `gh` CLI, Node.js 20, Next.js 16

**Pré-requisitos (verificar antes de começar):**
- `gh auth status` → deve mostrar conta autenticada
- `vercel whoami` → deve mostrar usuário logado
- `.env.local` disponível localmente com todos os valores reais

### Task 2: Criar workflow CI

**Files:**
- Create: `.github/workflows/ci.yml`

- [ ] **Step 1: Criar diretório e arquivo**

```bash
mkdir -p .github/workflows
```

Criar `.github/workflows/ci.yml` com o conteúdo abaixo:

```yaml
name: CI

on:
  pull_request:
    branches: [main, dev]

jobs:
  ci:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Lint
        run: npm run lint

      - name: Build
        run: npm run build
    env:
      NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY: ${{ secrets.NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY }}
      CLERK_SECRET_KEY: ${{ secrets.CLERK_SECRET_KEY }}
      NEXT_PUBLIC_CLERK_SIGN_IN_URL: /sign-in
      NEXT_PUBLIC_CLERK_SIGN_UP_URL: /sign-up
      NEXT_PUBLIC_CLERK_AFTER_SIGN_IN_URL: /
      NEXT_PUBLIC_CLERK_AFTER_SIGN_UP_URL: /
      NEXT_PUBLIC_CLERK_SIGN_IN_FALLBACK_REDIRECT_URL: /sign-in
      NEXT_PUBLIC_CLERK_SIGN_UP_FALLBACK_REDIRECT_URL: /sign-up
      ROTAS_BRASIL_TOKEN: ${{ secrets.ROTAS_BRASIL_TOKEN }}
      HERE_API_KEY: ${{ secrets.HERE_API_KEY }}
      TOMTOM_API_KEY: ${{ secrets.TOMTOM_API_KEY }}
```

- [ ] **Step 2: Commit e push**

```bash
git add .github/workflows/ci.yml
git commit -m "ci: add GitHub Actions lint and build workflow"
git push origin main
```

- [ ] **Step 3: Verificar que arquivo aparece no GitHub**

```bash

```

Navegar para a aba "Actions" — vai aparecer o workflow listado (sem runs ainda, só a definição).

### Task 5: Configurar branch protection em `main` e `dev`

**Files:**
- Nenhum arquivo — configuração via GitHub API

> Nota: o check name `ci` vem do campo `jobs.ci` definido no `ci.yml`. GitHub registra o nome do job como status check.

- [ ] **Step 1: Proteger branch `main`**

```bash
gh api \
  --method PUT \
  -H "Accept: application/vnd.github+json" \
  repos/ViniciusZimmerBrq/rotaz/branches/main/protection \
  --field required_status_checks='{"strict":true,"contexts":["ci"]}' \
  --field enforce_admins=false \
  --field required_pull_request_reviews='{"required_approving_review_count":0,"dismiss_stale_reviews":false}' \
  --field restrictions=null \
  --field allow_force_pushes=false \
  --field allow_deletions=false
```

Saída esperada: JSON com `url` contendo `/branches/main/protection`.
